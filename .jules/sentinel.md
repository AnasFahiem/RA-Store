## 2024-04-09 - [Insecure Randomness for File Uploads]
**Vulnerability:** The application was using `Math.random()` to generate unique file names for uploaded files. `Math.random()` is not cryptographically secure, which means its output is predictable. In the context of file uploads, predictable file names can lead to insecure direct object references (IDOR) or file overwriting vulnerabilities if an attacker can predict the generated file names.
**Learning:** `Math.random()` should never be used for generating sensitive random values, including unique identifiers, secrets, or file names where predictability is a security concern. It relies on a deterministic algorithm that is not designed to withstand cryptographic analysis.
**Prevention:** Always use cryptographically secure random number generators (CSPRNG) for such purposes. In Node.js, `crypto.randomUUID()` is the standard and secure way to generate universally unique identifiers (UUIDs) for file names and other identifiers.

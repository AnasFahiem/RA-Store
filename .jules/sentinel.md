## 2024-05-15 - [Critical] Server-Side Price Verification By-pass
**Vulnerability:** In `lib/actions/orderActions.ts` (`placeOrder`), the checkout mechanism trusted the `price` fields submitted by the client in the `items` array to calculate the order `total`. This allowed for Client-Side Price Manipulation (an Insecure Direct Object Reference) where a malicious user could submit arbitrary low prices or even negative prices.
**Learning:** Never trust the client for pricing information. Order totals must always be recalculated strictly on the server using authoritative records directly fetched from the database, completely ignoring client-provided pricing values for total accumulation.
**Prevention:** Query the `products` table using the submitted `productId`s, compute the true price (`dbPrice * quantity`), and explicitly assign and recalculate `item.price` based entirely on the `base_price` verified from the backend. Always return an error if a product ID is not found.

## 2024-05-20 - Missing Authentication on Admin Server Actions
**Vulnerability:** Several Server Actions in `lib/actions/hero.ts` and `lib/actions/header.ts` responsible for modifying site content (e.g., `addHeroSlide`, `deleteHeaderSlide`) lacked authentication and authorization checks while utilizing `createAdminClient()`. This created a Broken Access Control vulnerability allowing any unauthenticated external user to execute these administrative database operations.
**Learning:** Next.js Server Actions are essentially public API endpoints. If they use a service role client (`createAdminClient`) that bypasses Row Level Security (RLS), they must explicitly verify the user's session and role within the action itself before performing any operations.
**Prevention:** Always include `const session = await verifySession();` and check `session?.role` against allowed roles (`'admin'`, `'owner'`) at the beginning of any mutative Server Action, especially those interacting with the database via an admin client.

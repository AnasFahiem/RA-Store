## 2025-02-23 - [Authorization Bypass in createBundle]
**Vulnerability:** The `createBundle` server action in `lib/actions/bundleActions.ts` does not check if the user is an admin or owner before allowing them to create bundles of type `admin_fixed`. Any authenticated user, or even an unauthenticated user (since `session?.userId` can be null/undefined), can forge a request and create a bundle. Moreover, it accepts `priceOverride` which could be exploited by a malicious client.
**Learning:** Even though UI may hide functionality, server actions are public API endpoints. Not checking the user's role on the server means anyone can invoke it. Zod validation does not enforce authorization. `user_custom` bundles also need careful handling, but `admin_fixed` bundles definitely need an admin check. Also, `priceOverride` shouldn't be trusted for non-admin users.
**Prevention:** Always verify roles explicitly inside every server action that performs sensitive mutations, preferably using explicit role checks like `if (session?.role !== 'admin' && session?.role !== 'owner') { return { error: 'Unauthorized' }; }`.

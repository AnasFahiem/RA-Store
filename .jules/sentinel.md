## 2024-05-09 - Missing Role-Based Authorization in Server Actions
**Vulnerability:** Broken Access Control / RLS Bypass. Several Next.js Server Actions (`addHeroSlide`, `deleteHeroSlide`, `updateHeroSlideOrder`, `addHeaderSlide`, `deleteHeaderSlide`, `updateHeaderSettings`) were using `createAdminClient()` (which bypasses Row Level Security) without checking the user's role. This allowed any unauthenticated or unauthorized user to arbitrarily create, modify, or delete homepage content by invoking these public API endpoints directly.
**Learning:** Next.js Server Actions are exposed as public HTTP endpoints. Using `createAdminClient()` inside a Server Action creates a highly privileged operation. If explicit authorization checks are not implemented at the beginning of the action, the application trusts the client implicitly, leading to critical Broken Access Control vulnerabilities.
**Prevention:** Always use `verifyAdminAction()` (or equivalent strict role checks) at the very start of any Server Action that mutates state, retrieves sensitive data, or uses `createAdminClient()`. Never assume a Server Action is internal or protected simply because it is not directly linked in the UI for non-admin users.

## 2024-04-19 - Broken Access Control in Next.js Server Actions
**Vulnerability:** Several Server Actions (like `addHeroSlide`, `deleteHeaderSlide`, `createBundle`) were bypassing Supabase Row Level Security (RLS) by using `createAdminClient()`, which uses the service role key, *without* manually verifying the user's session role first. This allowed any unauthenticated or standard user to forge requests and perform administrative actions, such as mutating header settings or setting price overrides on bundles.
**Learning:** Next.js Server Actions are public API endpoints. When an action needs to bypass RLS (e.g., using a service role client to write to restricted tables), it MUST implement strict, manual authorization checks at the beginning of the function (e.g., verifying `session?.role === 'admin' || session?.role === 'owner'`).
**Prevention:** Always pair `createAdminClient()` with an explicit `getSession()` role check in user-facing Server Actions. Fail closed by returning a secure error object (e.g., `{ error: 'Unauthorized' }`) instead of throwing unhandled exceptions that might leak stack traces.

## 2025-04-30 - [Price Manipulation in Checkout]
**Vulnerability:** The server-side checkout action `placeOrder` calculated the order total by blindly trusting the `price` property of the `items` array sent by the client. A malicious user could submit a manipulated payload with item prices set to $0 or modify the quantities to get the order for free.
**Learning:** Never trust client-provided pricing or billing metrics. In Server Actions dealing with checkout or cart operations, the client should only provide product identifiers and quantities. Even if prices are provided by the client for display logic, they must be completely discarded when calculating the final authorized cost on the backend.
**Prevention:** Always perform a server-side authoritative fetch to get the current base prices of products directly from the database and perform cost calculations securely on the backend before finalizing any checkout or payment process. When items are grouped into bundles, calculate the price of bundles directly on the server to apply rules correctly.

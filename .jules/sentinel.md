## 2025-05-05 - Missing Authorization in Admin Server Actions
**Vulnerability:** Multiple critical Server Actions (`addHeroSlide`, `addHeaderSlide`, `uploadImage`, etc.) in Next.js lacked role-based authorization checks (`verifyAdminAction`) while using `createAdminClient()` (which bypasses Supabase RLS). Additionally, `uploadImage` lacked file extension validation and used predictable `Math.random()` filenames. `process.env.JWT_SECRET` was also unchecked at startup, risking empty signature keys.
**Learning:** Next.js Server Actions are public endpoints by default. Even if the UI is hidden behind an admin dashboard, the actions themselves can be called by anyone. Because Supabase Admin clients bypass Row Level Security, explicit backend authorization must be performed at the start of every admin-level action.
**Prevention:** Always use `verifyAdminAction()` (or equivalent explicit role checks) at the top of Server Actions before executing sensitive logic or database writes. Treat file uploads as untrusted input: explicitly validate extensions against a strict whitelist and use cryptographically secure methods like `crypto.randomUUID()` for file naming. Enforce environment variable checks (`JWT_SECRET`) at process initialization.

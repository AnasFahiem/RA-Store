## 2024-05-24 - [Missing Authorization Checks in Server Actions using Admin Client]
**Vulnerability:** Found multiple server actions (`createBundle`, `getPromoCodes`, `getPromoCodeById` in `lib/actions/bundleActions.ts`) that bypassed Row Level Security (RLS) by using `createAdminClient()`, but failed to manually verify the user's role before executing the query.
**Learning:** `createAdminClient()` explicitly bypasses Supabase RLS policies. Any server action utilizing it must enforce explicit role-based authorization (e.g., checking for 'admin' or 'owner' roles) at the beginning of the function. Failing to do so creates severe privilege escalation vulnerabilities, allowing unauthenticated or low-privilege users to modify admin data or read sensitive information like promo codes.
**Prevention:** Always pair `createAdminClient()` usage with a mandatory session verification block (`const session = await getSession(); if (session?.role !== 'admin' && session?.role !== 'owner') { return { error: 'Unauthorized' }; }`). Alternatively, use the centralized `checkAdminOrOwner()` utility if available to enforce this pattern uniformly and avoid SonarCloud code duplication errors.

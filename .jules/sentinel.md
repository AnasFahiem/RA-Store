## 2024-05-18 - [CRITICAL] Fix IDOR in Server Actions
**Vulnerability:** Next.js Server Actions modifying admin data (hero/header slides, header settings) lacked authorization checks while using `createAdminClient()`. `createAdminClient()` bypasses Row Level Security (RLS), meaning any user (including unauthenticated ones) could invoke these server actions directly to create, modify, or delete administrative content.
**Learning:** Functions that utilize a Supabase admin client (`createAdminClient`) inside Next.js Server Actions DO NOT implicitly inherit authorization from the application session or database RLS policies. They must explicitly verify the user's role prior to performing any database mutations.
**Prevention:** Always use `getSession()` or `verifySession()` and explicitly check the user's role (e.g., `if (session?.role !== 'admin' && session?.role !== 'owner') return { error: 'Unauthorized' };`) at the top of any server action that performs restricted operations, especially when using an admin database client.

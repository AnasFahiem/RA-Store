## 2026-03-31 - [Critical] Price Manipulation Vulnerability in Checkout
**Vulnerability:** The `placeOrder` server action calculated the total order price using client-provided `price` values from the checkout payload instead of authoritative database prices. Additionally, bundle logic vulnerabilities existed where unvalidated or unrelated items within a provided `bundleId` could bypass pricing checks if not strictly calculated.
**Learning:** Trusting client-side input for pricing calculations is a critical security flaw. Even when complex bundle logic is involved, every item must be explicitly validated against its server-side base price and bundle override definitions.
**Prevention:** Always recalculate order totals server-side by fetching authoritative prices from the database using provided product and bundle IDs. When iterating over client-provided bundled items, ensure items that fail bundle validation fall back to standard base pricing rather than being omitted from the total.

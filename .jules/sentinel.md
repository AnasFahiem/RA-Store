## 2024-05-14 - Broken Access Control in Server Actions and Price Manipulation
**Vulnerability:** Next.js Server Actions used to retrieve administrative data (`getPromoCodes`, `getPromoCodeById`, `getDiscountRuleById`) lacked explicit authorization checks, exposing sensitive data. Additionally, `createBundle` permitted users to set an arbitrary `priceOverride`, leading to a price manipulation vulnerability.
**Learning:** Next.js Server Actions are public API endpoints. When using a service-role Supabase client (`createAdminClient`), Row Level Security (RLS) is bypassed. Therefore, explicit role-based authorization must be enforced directly within the function using `getSession()`. Server-side validation of prices and overrides must strictly discard or reject values from unauthorized users.
**Prevention:** Always use explicit `session?.role === 'admin' || session?.role === 'owner'` checks when fetching sensitive data or allowing administrative changes (e.g. `price_override` and `admin_fixed` bundles) in Server Actions, especially when bypassing RLS.

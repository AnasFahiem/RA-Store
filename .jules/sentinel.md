## 2024-03-14 - [CRITICAL] Enforce JWT Secret Presence at Startup
**Vulnerability:** The application used `const key = new TextEncoder().encode(process.env.JWT_SECRET);` at the top level of `lib/auth/session.ts` without checking if `process.env.JWT_SECRET` was defined. If the environment variable was missing, `new TextEncoder().encode(undefined)` evaluates to an empty `Uint8Array(0)`, which causes `jose` to unpredictably crash when signing HMAC (HS256) JWTs or potentially allows creating zero-length secrets, resulting in insecure session token generation or denial-of-service (crashing the server on auth attempts).
**Learning:** In Node.js environments with recent versions, `new TextEncoder().encode(undefined)` does not throw an error but quietly returns a zero-length byte array. Missing environment checks for critical cryptographic keys create dangerous "fail-open" or unstable "fail-closed" scenarios.
**Prevention:** Always enforce explicit startup checks for cryptographic environment variables (like `JWT_SECRET`) and throw a descriptive `Error` if they are missing. This guarantees the application "fails securely" before accepting any authentication attempts.
